📖 Smart Expense Tracker — Process Step by Step

This document explains exactly what happens at every step a user takes in the app, and exactly how to deploy the full app (web + mobile) from zero to live.

PART 1 — USER FLOW: Step by Step

🔵 STEP 0 — User Opens the App

User opens browser → types your-app.vercel.app
          OR
User opens installed PWA from Android home screen
          OR
User opens React Native app from Play Store

What happens under the hood:

  • Browser/phone loads the React frontend from Vercel's CDN
  • React Router checks if the user has a valid JWT token in localStorage
  • If token exists and is not expired → goes directly to Dashboard
  • If no token → redirect to Login page
flowchart LR
    A[App Opens] --> B{JWT Token\nin localStorage?}
    B -->|Yes + Valid| C[Go to Dashboard]
    B -->|No / Expired| D[Go to Login Page]

🟢 STEP 1 — Registration / Login

1A. New User — Register with Email

User fills: Name | Email | Password → clicks "Sign Up"

What happens:

  1. React sends POST /api/auth/register to FastAPI backend
  2. FastAPI validates the data (Pydantic schema)
  3. Password is hashed with bcrypt — never stored as plain text
  4. New row created in users table in PostgreSQL (Supabase)
  5. A JWT access token (15 min) + refresh token (7 days) are created
  6. Tokens sent back to React → stored in localStorage
  7. User is redirected to Dashboard

1B. Existing User — Login

User enters Email + Password → clicks "Login"

What happens:

  1. POST /api/auth/login called
  2. FastAPI finds user by email in DB
  3. bcrypt.verify compares entered password with stored hash
  4. If match → new JWT token generated and returned
  5. React stores token → redirect to Dashboard

1C. Google OAuth Login (One-click)

User clicks "Continue with Google"

What happens:

  1. React redirects to GET /api/auth/google
  2. FastAPI redirects to Google's OAuth consent screen
  3. User approves → Google sends back a code to your backend
  4. FastAPI exchanges code for user's Google profile (name, email, photo)
  5. If user already exists → log in | If new → create account
  6. JWT token generated → returned to frontend
  7. Redirected to Dashboard ✅
sequenceDiagram
    participant U as User
    participant R as React App
    participant F as FastAPI
    participant G as Google OAuth
    participant DB as PostgreSQL

    U->>R: Click "Login with Google"
    R->>F: GET /api/auth/google
    F->>G: Redirect to consent screen
    G->>U: Shows permissions popup
    U->>G: Approves
    G->>F: Returns auth code
    F->>G: Exchange code → profile
    G->>F: {name, email, google_id}
    F->>DB: Find or create user
    DB->>F: User record
    F->>R: JWT Token
    R->>U: Redirect to Dashboard ✅

🟡 STEP 2 — Dashboard (Home Screen)

After login, user lands on the Dashboard. This is loaded by calling:

GET /api/dashboard/summary     → total spent this month, income, savings
GET /api/dashboard/breakdown   → % split by category (Food 40%, Travel 20%...)
GET /api/dashboard/trend       → day-by-day spending for this month
GET /api/insights              → AI-generated insights (loaded separately)
GET /api/budgets               → budget limits and % consumed

What the user sees:

  • 💳 Total spent this month card
  • 📊 Pie chart: category breakdown
  • 📈 Bar chart: daily spending trend
  • 🏦 Financial Health Score (0–100)
  • 💡 AI Insight cards ("You spent 25% more on food this week")
  • ⚠️ Budget warning banners if any category > 80%
All chart data comes from the backend which queries PostgreSQL, aggregates with SQL, and returns JSON.

🔴 STEP 3 — Adding an Expense (4 Ways)

WAY 1 — Manual Entry (Simplest)

User clicks "+" button → fills form:
  Amount: 350
  Category: Food
  Date: Today
  Note: Dinner at Barbeque Nation
→ clicks "Save"

What happens:

  1. React sends POST /api/expenses with the form data
  2. FastAPI validates with Pydantic (ensures amount > 0, date is valid, etc.)
  3. If category is left blank → AI auto-categorization kicks in (see Step 4)
  4. New row inserted into expenses table in PostgreSQL
  5. Response sent back → Dashboard re-fetches and updates charts

WAY 2 — OCR Receipt Scanner (Photo of Bill)

User clicks "Scan Receipt" → takes photo of bill
→ uploads image

What happens step by step:

flowchart TD
    A[User takes photo of bill] --> B[React sends image\nPOST /api/expenses/ocr]
    B --> C[FastAPI receives image file]
    C --> D[Image pre-processed\nOpenCV grayscale + threshold]
    D --> E[Tesseract OCR runs\nextracts raw text]
    E --> F{OCR text quality\ngood enough?}
    F -->|Yes| G[Send text to Gemini API]
    F -->|No| H[Return error: Image unclear\nAsk user to retake]
    G --> I[Gemini extracts:\nAmount, Merchant, Date, Category]
    I --> J[Return prefilled form to user]
    J --> K[User confirms/edits data]
    K --> L[POST /api/expenses → save to DB]
    L --> M[Dashboard updates ✅]

What Gemini does with OCR text:

OCR raw text: "SWIGGY ORDER TOTAL Rs. 485 DATE 04-04-2026 ORDER#12345"

Gemini prompt: "Extract: amount, merchant, date, category from this text"

Gemini output: {
  "amount": 485,
  "merchant": "Swiggy",
  "date": "2026-04-04",
  "category": "Food & Dining"
}

User sees a prefilled form → confirms → saved ✅

WAY 3 — 🏦 Bank Statement Import ⚡ Advanced Feature

👆 What is this feature?

Instead of adding expenses one-by-one, you can upload your entire bank/UPI statement and the app will automatically import all your transactions — categorized and ready in seconds.

📊 1 upload = 1 month of expenses added instantly

💡 This is labeled as an "Import from Bank (Advanced)" option inside the app — it won't confuse regular users, but power users will love it!

📥 STEP 1 — How to Download Your Bank Statement

Before uploading to the app, you need to download your statement from your bank or UPI app. Here's how:

📱 Google Pay

  1. Open the Google Pay app
  2. Tap your profile photo (top right)
  3. Go to Manage Google Pay account
  4. Tap Transaction history
  5. Tap Download statement → choose month
  6. Select CSV format → Save to phone

📱 PhonePe

  1. Open the PhonePe app
  2. Tap History (bottom menu)
  3. Tap the filter / download icon (top right)
  4. Select date range (e.g., last 1 month)
  5. Tap Download Statement
  6. Choose PDF or CSV → Save

📱 Paytm

  1. Open the Paytm app
  2. Tap Passbook from the home screen
  3. Tap Transactions
  4. Tap the ↓ Download icon (top right)
  5. Select month → tap Download PDF

🏦 SBI YONO / Internet Banking

  1. Open YONO SBI app or go to onlinesbi.sbi
  2. Login → go to My Accounts
  3. Select your account → tap Account Statement
  4. Choose date range (From Date → To Date)
  5. Click Download as CSV or PDF

🏦 HDFC Mobile Banking / NetBanking

  1. Open HDFC Mobile Banking app or visit hdfcbank.com
  2. Login → tap Accounts
  3. Select account → tap View Statement
  4. Set date range → tap Download
  5. Select Excel / CSV format → Save

In the app → click "Import from Bank (Advanced)"
→ drag & drop your downloaded PDF or CSV file
→ app automatically reads and imports all transactions

What happens behind the scenes:

    C -->|PDF| D[pdfplumber extracts\ntables from PDF pages]

                          ↓  Auto saved
Result:     Expense added to your dashboard ✅

🟣 STEP 4 — AI Auto-Categorization (Runs in background)

Every time an expense is added without a category, or imported from a statement:

flowchart LR
    A[Transaction description] --> B[NLP Categorizer]
    B --> C{Method}
    C -->|Keywords first| D[Rule engine:\nSwiggy→Food\nUber→Transport\nNetflix→Entertainment]
    D -->|Matched| E[Use keyword category]
    D -->|Not matched| F[Send to Gemini API]
    F --> G[Gemini classifies\nwith confidence score]
    G --> E
    E --> H[Save category to DB]

Categories recognized:

Food & Dining | Transport | Shopping | Entertainment | Bills & Utilities | Healthcare | Education | Travel | Investments | Other

🟠 STEP 5 — AI Insights Engine

Every night at midnight (Celery scheduled task):

flowchart TD
    A[Celery Scheduler\ntriggers nightly] --> B[Fetch last 30 days expenses\nfor each user]
    B --> C[Run 4 analysis modules]
    C --> D[Module 1: Spending Patterns\nweekday vs weekend, morning vs night]
    C --> E[Module 2: Category Trends\ncompare to last month %]
    C --> F[Module 3: Anomaly Detection\nIsolation Forest on amounts]
    C --> G[Module 4: Prediction\nProphet forecasts next month]
    D --> H[Generate natural language insights\nvia Gemini API]
    E --> H
    F --> H
    G --> H
    H --> I[Store in insights table]
    I --> J[Send push notification\nvia Firebase FCM]
    J --> K[User sees insight card\non next Dashboard visit]

Example Insights generated:

  • 💡 "Your Food spending is ₹3,200 this month — 28% higher than last month"
  • ⚠️ "Unusual transaction: ₹12,000 at Amazon on Apr 3 — is this correct?"
  • 🔮 "Based on your pattern, you'll spend ~₹18,500 next month"
  • 💰 "You could save ₹1,800/month by reducing Swiggy orders by 30%"

🔵 STEP 6 — Financial Health Score

GET /api/insights/health  →  returns score 0–100

How the score is calculated:

flowchart TD
    A[Financial Health Score] --> B[3 Components]
    B --> C["Savings Ratio 40pts\nIncome - Expenses / Income\n>30% = 40pts\n>10% = 20pts\n<10% = 5pts"]
    B --> D["Budget Adherence 35pts\nHow many categories within limit\n100% within = 35pts\n80% = 25pts\n<50% = 10pts"]
    B --> E["Spending Stability 25pts\nStd deviation of daily spending\nLow variance = 25pts\nHigh variance = 5pts"]
    C --> F[Total Score / 100]
    D --> F
    E --> F
    F --> G[Score 80-100: Excellent 🟢]
    F --> H[Score 60-79: Good 🟡]
    F --> I[Score 40-59: Fair 🟠]
    F --> J[Score below 40: Needs Work 🔴]

🟤 STEP 7 — Budget Alerts

User sets budget: Food = ₹5,000/month

What happens when budget is nearing:

  1. Every time a new expense is saved → Celery task checks budget
  2. SUM(expenses WHERE category='Food' AND month=current) vs limit
  3. If > 80% → send push notification via Firebase FCM
  4. If > 100% → send "LIMIT EXCEEDED" alert
  5. Alert stored in DB so it's not sent twice (alert_sent = true)

⚪ STEP 8 — AI Chat Assistant

User types: "How much did I spend on food last month?"
         or "Where can I save money?"
         or "Show me my top 5 expenses this week"

What happens (RAG — Retrieval Augmented Generation):

sequenceDiagram
    participant U as User
    participant R as React Chat UI
    participant F as FastAPI
    participant DB as PostgreSQL
    participant G as Gemini API

    U->>R: "How much on food last month?"
    R->>F: POST /api/chat {message}
    F->>DB: Query: SELECT SUM(amount) WHERE category='Food'\nAND month=March
    DB->>F: ₹6,840
    F->>G: "User spent ₹6840 on food in March.\nAnswer: How much did I spend on food last month?"
    G->>F: "You spent ₹6,840 on Food in March. That's your highest category!"
    F->>R: Response text
    R->>U: "You spent ₹6,840 on Food in March 🍕"
The AI doesn't guess — it first queries your real data from DB, then asks Gemini to phrase the answer naturally. This is called RAG (Retrieval Augmented Generation).

# backend/services/notification.py
import firebase_admin
from firebase_admin import messaging

def send_budget_alert(user_fcm_token, category, percentage):
    message = messaging.Message(
        notification=messaging.Notification(
            title="⚠️ Budget Alert",
            body=f"You've used {percentage}% of your {category} budget!"
        ),
        token=user_fcm_token

DEPLOY STEP 10 — GitHub Actions CI/CD (Auto Deploy)

Create .github/workflows/deploy.yml:

yaml
name: Deploy Smart Expense Tracker

on:
  push:
    branches: [main]

jobs:
  test-backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with: { python-version: '3.11' }
      - name: Install & Test
        run: |
          cd backend
          pip install -r requirements.txt
          pytest tests/ -v

  deploy-backend:
    needs: test-backend
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to Render
        run: curl -X POST ${{ secrets.RENDER_DEPLOY_HOOK }}

  deploy-frontend:
    needs: test-backend
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.ORG_ID }}
          vercel-project-id: ${{ secrets.PROJECT_ID }}

What this does:

  • Every time you git push to main branch:

1. ✅ Run all backend tests

2. If pass → auto-deploy backend to Render

3. If pass → auto-deploy frontend to Vercel

  • Zero manual deployment needed after setup ✅
